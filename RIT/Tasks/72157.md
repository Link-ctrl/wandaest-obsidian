В ходе анализа выяснилось, чтопричиной большого размера исполняемого файла tag (4.2 Мб) являются некорректная сборка библиотеки boost: библиотека компилируется целиком, а также отсутствие каких-либо оптимизационных флагов CMake.
Были предприняты следующие меры:

Шаг 1: Оптимизация boost

1. /bootstrap.sh --with-toolset=gcc
   Генерирует b2 (Boost.Build tool) и config.jam для сборки 
2. Настройка toolset в project-config.jam внутри boost: Добавил строку using gcc : arm : arm-linux-gnueabi-g++ 
   Определяет custom toolset для кросс-компиляции
3. ./b2 toolset=gcc-arm link=static runtime-link=static cxxflags="-march=armv5tej -marm -Os" cflags="-march=armv5tej -marm -Os" --with-system --with-container --with-random --with-stacktrace stage
   Непосредственно собирает boost с необходимыми опциями и модулями, собранные библиотеки хранятся в boost/stage/lib

Шаг 2: Оптимизация CMake
1. ```cmake
set(CMAKE_CXX_FLAGS "-Wall --static -marm -march=armv5tej -Os -ffunction-sections -fdata-sections -flto")
set(CMAKE_EXE_LINKER_FLAGS "${CMAKE_EXE_LINKER_FLAGS} -Wl,--gc-sections -flto")
   ```
- CMAKE_CXX_FLAGS: Компилятор флаги. -Os: Оптимизация для размера (упрощает код, -20–30%). -ffunction-sections/-fdata-sections: Функции/данные в отдельные секции для удаления. -flto: Link-time opt (межмодульное удаление). -
- CMAKE_EXE_LINKER_FLAGS: Линкер. -Wl,--gc-sections: Garbage collection неиспользуемых секций.
2. ```cmake
include_directories(boost_1_80_0)
link_directories(boost_1_80_0/stage/lib)
set(Boost_USE_STATIC_LIBS ON) 
   ```
   - Прямая линковка boost без find_package
 Шаг 3: Изменения в build_app.sh
 ```bash
 #!/bin/bash
cmake -DCMAKE_BUILD_TYPE=MinSizeRel -DCMAKE_MAKE_PROGRAM=make -DARM_BUILD=ON -DUSE_GNSS=OFF -S $PWD -B $PWD/cmake-build-arm
cmake --build $PWD/cmake-build-arm --target tag -j 1
arm-linux-gnueabi-size $PWD/cmake-build-arm/tag
 ```
- -DCMAKE_BUILD_TYPE=MinSizeRel: Тип сборки для размера (активирует -Os global).
- arm-linux-gnueabi-size $PWD/cmake-build-arm/tag анализ размера

В результате удалось добиться сокращения размера с 4,2 Мб до 2,3 Мб (1,5 Мб на теге). Потенциально возможно сокращения размера до 0,5-1 Мб, что требует перехода на актуальную версию buildroot, переноса сборки исполняемого файла tag в buildroot, а также перехода со стандартной библиотеки C ulibc на musl, что выходит за рамки данного кейса.
